Доставка кода
после миграции надо написать доставщика для корп ноута, гигачат уже предложил код.
Таким образом последняя версия кодов всегда будет находиться на стороне разработки тут в vscode, а в линукс - доставка.
Здесь файлы последних версий будут без указания версии в именах, чтобы линукс не видел разницы менжду версиями при запуске.
Файлы устаревших версий переименовываются сразу добавлением версии и перемещаются в history.
Резюмируя, в корне проекта лежат все файлы новейших версий без упоминания номера в имени.


Да именно на сервере создавать объект conninfo (а не currentdb)
он должен содержать инфо о:
- хосте сервера
- порте сервера
- подключённом к БД юзере
- имя текущей базы
И передавать после каждой команды от клиента ему обратно весь этот объект целиком. После чего на клиенте пересматривается приглашение.

Приглашение должно быть такого вида:
username@ip:port/dbname>
pdba@10.0.1.22:2345/database1>

насколько хороша библиотека sqlparse?
Написать функции выполнения запросов в таком порядке:
1) парсинг и выполнение CREATE DATABASE
2) парсинг и выполнение CREATE SCHEMA
3) парсинг и выполнение CREATE TABLE
4) парсинг и выполнение SELECT (наверное сложнее всего)
