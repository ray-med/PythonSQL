CONNINFO
Да именно на сервере создавать объект conninfo (а не currentdb)
он должен содержать инфо о:
- хосте сервера
- порте сервера
- подключённом к БД юзере
- имя текущей базы
И передавать после каждой команды от клиента ему обратно весь этот объект целиком.
После чего на клиенте пересматривается приглашение.

Приглашение должно быть такого вида:
username@ip:port/dbname>
pdba@10.0.1.22:2345/database1>

насколько хороша библиотека sqlparse?
Написать функции выполнения запросов в таком порядке:
1) парсинг и выполнение CREATE DATABASE
2) парсинг и выполнение CREATE SCHEMA
3) парсинг и выполнение CREATE TABLE
4) парсинг и выполнение SELECT (наверное сложнее всего)

Доставка кода
Последняя версия кодов всегда будет находиться на стороне разработки тут в vscode, а в линукс - доставка.
Файлы последних версий будут в корне репозитория без указания версии в именах - для удобства.
Файлы устаревших версий переименовываются добавлением версии и перемещаются в history.
